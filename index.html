Defense in Depth

In traditional network architecture, the access layer is the edge of the network where end devices — PCs, phones, printers, and IoT devices — connect to the switches. Because it is the point of entry for users, both legitimate and malicious, it is highly vulnerable to Layer 2 attacks.

To protect the network infrastructure, administrators use a variety of access layer threat mitigation techniques. Below are the most common and effective methods, categorized by the threats they prevent. Securing the access layer requires a Defense-in-Depth approach: Port Security and 802.1X keep bad actors off the physical ports; DHCP Snooping, DAI, and IPSG stop bad traffic if an actor manages to connect; and BPDU Guard and DTP mitigation protect the underlying architecture of the switches themselves.

01
Category 1

Identity & Device Control

Ensure that only authorized devices and users can connect to the network.

🔒
Port Security
⚡ Mitigates: MAC Flooding & Unauthorized Devices
What it does: Restricts a switch port so that only a specific MAC address (or a limited number of MAC addresses) can send traffic through it.
Mechanism

Restricts port access to specific MAC addresses. If an unauthorized MAC connects, the switch takes action based on the configured violation mode:

Protect Drops unauthorized traffic silently.
Restrict Drops traffic and logs the violation.
Shutdown Disables the port entirely (err-disabled).
🛡️
802.1X Authentication (NAC)
⚡ Mitigates: Rogue Devices & Lateral Movement
What it does: Provides port-based network access control using an authentication server (usually RADIUS/TACACS+).
Mechanism

When a device plugs in, the port remains in an unauthorized state — blocking all traffic except authentication protocols — until the user or device proves their identity using credentials or a digital certificate. Only then is full network access granted.

02
Category 2

Layer 2 Protocol Security

Attackers abuse basic networking protocols (DHCP and ARP) to intercept traffic or cause denial-of-service.

🌐
DHCP Snooping
⚡ Mitigates: DHCP Starvation & Rogue Gateways
What it does: Prevents rogue DHCP servers from assigning invalid IP addresses to clients, stopping Man-in-the-Middle attacks via rogue default gateways.
Mechanism

Switch ports are configured as Trusted (connected to the legitimate DHCP server) or Untrusted (connected to end users). DHCP offer packets are only allowed on trusted ports.

Key Feature

Builds a DHCP Snooping Binding Database mapping MAC addresses to assigned IP addresses — vital for DAI and IPSG.

🔍
Dynamic ARP Inspection (DAI)
⚡ Mitigates: ARP Spoofing & MitM Attacks
What it does: Validates ARP packets in the network to prevent attackers from spoofing MAC-to-IP relationships.
Mechanism

Intercepts all ARP requests/replies on untrusted ports and checks them against the DHCP Snooping Binding Database. If the MAC-to-IP mapping doesn't match the database, the packet is dropped immediately.

🚧
IP Source Guard (IPSG)
⚡ Mitigates: IP Spoofing & Policy Bypass
What it does: Prevents IP spoofing by restricting the IP addresses a device can use on a port.
Mechanism

Like DAI, it uses the DHCP Snooping database. It applies a dynamic ACL to the port, ensuring the device can only send traffic using the exact IP address assigned to it by the legitimate DHCP server.

03
Category 3

VLAN Security

VLANs separate broadcast domains, but attackers can use VLAN Hopping to access secure networks from a less secure one.

🚫
Disabling DTP (Dynamic Trunking Protocol)
⚡ Mitigates: VLAN Hopping via Switch Spoofing
What it does: Prevents "switch spoofing" where an attacker's laptop pretends to be a switch to form a trunk link, granting access to all VLANs.
Mechanism

Administrators explicitly configure user-facing ports as static access ports (switchport mode access) and disable DTP negotiation (switchport nonegotiate), preventing attackers from establishing a trunk link.

🏷️
Native VLAN Management
⚡ Mitigates: Double-Tagging VLAN Hopping
What it does: Prevents "double-tagging" attacks, where an attacker encapsulates malicious traffic with two VLAN tags to bypass routing restrictions.
Mechanism

The native VLAN should be changed from the default (VLAN 1) to an unused, "dead" VLAN. Furthermore, native VLANs should not be assigned to any user access ports, eliminating the double-tagging attack vector.

04
Category 4

Spanning Tree Protocol (STP) Protection

STP prevents loops in the network, but attackers can manipulate it to intercept traffic or crash the network.

⛔
BPDU Guard
⚡ Mitigates: Unauthorized Switches & Network Loops
What it does: Protects edge ports (where computers and phones connect) from receiving STP Bridge Protocol Data Units (BPDUs).
Mechanism

If an access port receives a BPDU — meaning someone plugged a switch into a wall jack — the port is immediately put into an err-disabled (shutdown) state, preventing accidental or malicious network topology changes.

👑
Root Guard
⚡ Mitigates: STP Manipulation & MitM via Root Bridge
What it does: Prevents an unauthorized switch from becoming the "Root Bridge" of the STP topology.
Mechanism

If a port configured with Root Guard receives a superior BPDU, it places the port into a root-inconsistent (blocking) state until the malicious BPDUs stop — preventing all network traffic from being routed through an attacker-controlled switch.

Category 5

Administrative Best Practices

Beyond specific protocols, standard physical and administrative controls are critical at the access layer.

🔌
Shutdown Unused Ports
Any switch port not actively in use should be administratively shut down using the shutdown command, preventing physical connection by unauthorized individuals.
🕳️
Black-hole / Parking Lot VLANs
Unused ports should be assigned to an isolated VLAN that has no routed access to the rest of the network or the internet, containing any potential threats.
🔐
Physical Security
Securing wiring closets and server rooms ensures attackers cannot easily plug directly into core or distribution switches, complementing all logical security measures.
Summary

Defense-in-Depth

A layered approach ensures that multiple security controls must be defeated before an attacker can succeed.

LAYER 1
Keep Bad Actors Off Physical Ports
Port Security & 802.1X Authentication prevent unauthorized physical connections.
LAYER 2
Stop Bad Traffic If Actor Connects
DHCP Snooping, Dynamic ARP Inspection (DAI), and IP Source Guard (IPSG) neutralize Layer 2 attacks in flight.
LAYER 3
Protect VLAN Architecture
Disabling DTP and Native VLAN management prevent VLAN hopping and unauthorized lateral movement across segments.
LAYER 4
Protect Underlying Switch Architecture
BPDU Guard and Root Guard secure the STP topology from manipulation and network loop attacks.
LAYER 5
Physical & Administrative Controls
Shutdown unused ports, parking lot VLANs, and physical security of wiring closets complete the defense.
